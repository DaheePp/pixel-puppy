## 1. 개요

### 1.1 제품 비전
컴퓨터 모니터 위를 강아지 캐릭터가 네발로 자유롭게 돌아다니며 혼자 놀고, 유저가 가끔 밥을 주거나 산책시키거나 놀아주는 **방치형 데스크톱 펫** 서비스.

### 1.2 핵심 콘셉트
- **방치형**: 유저가 항상 지켜보지 않아도 강아지가 스스로 생활함. 컴퓨터 배경에 켜두는 용도이므로 스탯은 천천히 변화
- **가끔의 돌봄**: 하루 몇 번의 가벼운 인터랙션(밥/놀이/산책/쓰다듬기)으로 유대감 형성
- **성장**: 잘 돌볼수록 레벨이 오르고 강아지가 커짐

### 1.3 단계별 전략
| 단계 | 형태 | 상태 |
|---|---|---|
| 1단계 | 브라우저 프로토타입 (단일 HTML) | ✅ 완료 |
| 2단계 | 웹 배포 (GitHub Pages) | ✅ 라이브 |
| 3단계 | 데스크톱 앱 (Electron) — 실제 바탕화면 위를 돌아다님 | 🔜 예정 |

## 2. 타깃 & 사용 시나리오

- 업무 중 컴퓨터 화면 한켠에 켜두고 정서적 환기를 원하는 직장인
- 실제 반려동물을 키우기 어려운 환경의 유저

**핵심 시나리오**: 출근 → 강아지 탭 열어두기 → 일하다가 가끔 보면 강아지가 혼자 걷고, 장난감 갖고 놀고, 졸리면 쿠션에서 잠 → 점심에 밥 주고 → 오후에 잠깐 공놀이/산책 → 퇴근

## 3. 기능 명세

### 3.1 강아지 자율 행동 (방치 시)
- 화면 전체를 바닥 삼아 2D로 자유 이동 (아래쪽일수록 크게 보이는 원근감)
- 행동 종류: 걷기, 멈춰서 두리번, 앉기, 엎드리기, 장난감 공 갖고 혼자 놀기
- 에너지가 12 이하로 떨어지면 스스로 쿠션으로 가서 잠 (85까지 차면 기상)
- 상태에 따라 혼잣말 말풍선 (배고프면 "배고파요...", 심심하면 "같이 놀아요!")
- 꼬리 흔들기, 걷기 애니메이션(다리 교차), 잘 때 눈 감기

### 3.2 스탯 시스템
| 스탯 | 감소 속도 | 회복 수단 |
|---|---|---|
| 🍖 배고픔 | 3시간에 100→0 + 놀이 -8, 산책 -12 | 밥주기 +50 |
| 💗 행복 | 3시간에 100→0 | 놀이 +35, 산책 +30, 밥 +5, 쓰다듬기 +4, 혼자놀기 +3 |
| ⚡ 에너지 | 6시간에 100→0 + 놀이 -8, 산책 -12 | 잠 (약 3분에 완충), 오프라인 시 2분당 +1 |

- 자리 비운(오프라인) 시간만큼 스탯 감소 적용. 단 **최저 5 보호** (죽음/가출 등 부정적 결말 없음)
- 스탯 부족 시 행동 제한: 에너지 10 이하 놀이 불가, 20 이하 산책 불가, 배고픔 90 이상 밥 거부

### 3.3 유저 인터랙션

#### 🍖 밥주기
- 바닥 임의 위치에 밥그릇 등장 → 강아지가 달려가서 4초간 먹음
- **식사 중에는 모든 버튼 잠금** (유일한 잠금 상태). 버튼 아래 프로그레스바로 남은 시간(이동+식사) 표시

#### ⚽ 놀아주기
- 공이 등장해 굴러감 → 강아지가 쫓아감
- **유저 인터랙션**: 공을 드래그해서 끌다가 놓으면 손의 속도로 던져짐(플링). 강아지가 공을 갖고 있어도 뺏을 수 있음 (공이 항상 위 레이어 + 히트 영역 확대)
- **혼자 놀기 AI**: 공에 도착하면 ①코로 톡 쳐서 굴리고 쫓아가기(45%) ②공을 코앞에 붙이고 어슬렁 몰고 다니기(35%) ③신나서 폴짝(20%)
- **종료**: 유저가 10초간 공을 안 건드리면 자동 종료 (전용 종료 버튼 없음)

#### 🦮 산책
- 집 테마에 따라 **산책지로 배경 전환**: 잔디→숲속, 해변→해변도로, 거실→공원
- 산책지에서 집 소품(개집/쿠션 등)은 숨김
- **유저 인터랙션**: 바닥 터치 시 그 위치로 달려옴(빠른 속도 230) + 산책 10초 연장. 혼자 거닐 땐 느긋한 속도(160)
- 무반응 10초면 집으로 복귀

#### 💤 재우기
- 강아지가 쿠션으로 이동(속도 120) 후 수면. 자는 동안 버튼 아래 에너지 회복 진행률 표시
- 에너지 85 이상이면 거부 ("지금은 안 졸려요!")
- 자는 중 강아지 클릭하면 기상

#### 🐾 부르기 / 쓰다듬기
- 평상시 배경 클릭 → 발자국 이펙트 + 강아지가 달려와서 앉아 기다림
- 강아지 클릭 → 쓰다듬기 (하트 이펙트, 행복 +4)
- 자는 중 부르면 "음냐... 불렀어요?" 하고 일어나서 옴

#### 행동 전환 규칙
- 식사 중에만 다른 행동 차단. 놀이/산책/잠/혼자놀기 중에는 다른 버튼을 누르면 하던 활동을 정리(`interrupt`)하고 즉시 전환
- 놀이/산책 도중 전환해도 행복 보상은 지급

### 3.4 시작 경험 (온보딩)
- 첫 실행 시 **견종 선택**: 갈색 푸들 / 흰색 말티즈 / 포메라니안 / 웰시코기 / 시바견 (미리보기 제공)
- 이름 짓기 (기본값 "콩이", 이후 이름 클릭으로 변경 가능)

### 3.5 공간 & 테마
- **집 테마 3종** (패널에서 전환, 저장됨): 🌿 잔디밭 / 🏖 해변 / 🏠 거실
- 테마별 바닥 시작 높이 다름 (해변은 바다 아래, 거실은 벽 아래)
- **소품**: 개집, 쿠션(수면 장소), 뼈다귀, 장난감 공 — 강아지가 생활하는 공간 느낌
- 모든 배경 장식은 **이모지 금지, 픽셀 아트로 직접 제작** (꽃/풀/조개/불가사리/야자수/창문/화분/나무/버섯/벤치)

### 3.6 성장 시스템
- 경험치: 산책 +20, 놀이 +15, 밥 +10, 쓰다듬기 +2
- 레벨업 필요 XP = 레벨 × 50, 레벨이 오르면 몸이 조금씩 커짐 (Lv.13에서 최대 1.5배)
- ⚠️ 현재 레벨 상한/최종 보상 없음 → 개선 백로그

### 3.7 저장
- localStorage 자동 저장 (10초 주기 + 창 닫을 때)
- "처음부터 다시 키우기"로 완전 초기화 (견종 선택부터 재시작)

## 4. UX 원칙 (기획 중 확정한 규칙)

1. 말풍선은 **강아지의 말**만. 조작 안내는 패널 하단 안내 문구 영역에 표시
2. 프로그레스바는 정보가 의미 있을 때만: 식사 남은 시간 / 수면 회복률 / 스탯 부족 시 회복률. 유저 인터랙션으로 연장되는 활동(놀이/산책)에는 미노출
3. 버튼 잠금은 최소화 — 식사 중에만 잠금, 나머지는 즉시 전환
4. 부정적 결말 없음 (스탯 최저 5 보호)
5. 픽셀 아트 톤 통일 (¾ 시점 얼굴, 눈 반짝이, 볼터치 / 이펙트용 이모지 ❤️✨🐾만 예외 허용)

## 5. 기술 개요

- **단일 HTML 파일** (의도적 설계 — 배포 = 파일 1개 업로드). 프레임워크/라이브러리 없음
- 60fps 게임 루프 + 상태 머신, Canvas 픽셀 렌더링 (코드 컨벤션은 CLAUDE.md 참고)
- 견종은 데이터(`BREEDS`)로 정의 — 새 견종 추가가 쉬움
